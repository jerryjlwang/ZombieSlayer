Introduction to Retrieval-Augmented Generation

Retrieval-Augmented Generation (RAG) combines a parametric language model with a non-parametric retrieval component. At inference time, relevant documents are fetched from an external store and prepended to the prompt, grounding the model's output in up-to-date or domain-specific knowledge.

The retrieval component is typically a dense vector index (FAISS, Pinecone, Weaviate) queried via approximate nearest-neighbor search. Documents are chunked, embedded with a bi-encoder, and indexed offline. At query time the question is embedded with the same encoder and the top-k chunks retrieved.

Key failure modes include retrieval of irrelevant chunks, context window overflow when k is large, and stale indexes. Hybrid retrieval — combining BM25 sparse matching with dense embeddings — often outperforms either approach alone.
